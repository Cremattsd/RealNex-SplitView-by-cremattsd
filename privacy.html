Privacy Policy for Goose Maverick

Last Updated: November 2025

The Goose Maverick Chrome Extension (“the Extension”) is developed and maintained by Matthew Smith (CREmattSD). This Privacy Policy explains how user information is handled in connection with this Extension.

1. Data Collection

The Goose Maverick Chrome Extension does not collect, store, transmit, or share any personal or sensitive user data. It operates entirely within your local browser environment.

  • No personally identifiable information (e.g., name, email, or address) is collected.
  • No authentication credentials or payment details are accessed or stored.
  • No browsing history, keystrokes, or user communications are monitored.
  • No external servers or databases are used for data storage or tracking.

2. Purpose and Functionality

The Extension’s sole purpose is to enhance the RealNex web experience by enabling visual themes (light/dark mode, glass interface) and UI optimizations. These changes are applied locally to pages viewed in your browser and do not affect site functionality or data.

3. Permissions Explanation

The following permissions are requested solely to fulfill the Extension’s single purpose:

  • storage – to save UI preferences such as dark mode state and layout settings locally.
  • scripting – to inject style and logic scripts for page customization.
  • activeTab – to apply changes only to the active tab when toggled by the user.
  • host permissions – limited to RealNex.com and related domains to modify UI elements only.

4. No Remote Code or Tracking

The Extension contains no remote scripts, eval functions, or dynamic imports of third-party code. It does not use analytics, cookies, or tracking pixels.

5. Third Parties

Goose Maverick does not sell, rent, or share user information with any third parties. The Extension does not interact with or send information to any external APIs, services, or domains beyond those specified in the manifest.

6. Data Security

Since no personal data is collected or transmitted, there are no associated security risks to user data. All operations occur locally within the Chrome browser sandbox.

7. Updates to this Policy

If future updates introduce new functionality requiring data handling, this Privacy Policy will be updated to clearly describe those changes before release.
